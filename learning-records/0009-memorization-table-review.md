# 0009 — 暗記表レビュー：カバー範囲とレッスンへの導線

## きっかけ

ユーザーからの質問：「暗記表は妥当か？これを全部暗記していたらとりあえず通るレベルになっているか？その暗記表に、わからなければこのレッスンで解説、みたいに誘導ができているか？」

## レビュー結果（修正前）

`reference/numbers-to-memorize.html` を各レッスンの試験数値と突き合わせた結果、**レッスンのクイズで問われているのに暗記表に載っていない数値**が複数見つかった。

| 抜けていた数値 | 出典レッスン | リスク |
|---|---|---|
| 漏電遮断器（ELB）30mA以下・0.1秒以内（水気ある場所15mA） | レッスン7 | D種接地の緩和条件を理解する前提知識が欠落 |
| 電圧降下係数 35.6 / 17.8 / 30.8 | レッスン14 | 計算問題（配点約12%の科目②）の核 |
| 分岐回路の3点セット（15A〜50A、遮断器・電線太さ・コンセント） | レッスン15 | 「20Aヒューズだけ2.0mm」のひっかけが未収録 |
| PSEマーク（菱形／円形） | レッスン11 | 単独の暗記問題として出題されうる |
| 認定電気工事従事者の取得条件（3年実務／認定講習） | レッスン11 | 第2種のその先の知識だが法令科目に絡む |

またレッスン9・10・15は「このレッスンで終わっていないこと」の文中で `numbers-to-memorize.html` を**数値の総復習先として名指し**していたが、実際にはそのレッスンの数値が載っていない、という内部矛盾があった。

さらに、暗記表・`symbol-decoder.html`・`cable-abbreviations.html` はいずれもページ最下部に汎用の `lesson-nav` しか持たず、**見出しごとに「わからなければどのレッスンに戻るか」が示されていなかった**。ホバーで答えを見て理屈を忘れていても、該当レッスンへワンクリックで戻れない状態だった。

## 修正内容

1. `numbers-to-memorize.html` に5つの新セクションを追加（ELBの感度電流・電圧降下係数・分岐回路3点セット・PSEマーク・認定電気工事従事者の条件）。すべて `reveal-table` 形式でホバー暗記に統一。
2. 全見出し（`<h2>`）の直下に `<p class="see-lesson">わからなければ → <a href="...">レッスンXで解説</a></p>` を追加。対象：`numbers-to-memorize.html`（9箇所）、`symbol-decoder.html`（4箇所）、`cable-abbreviations.html`（2箇所）。
3. `assets/reveal-answer.css` に `.see-lesson` クラスを新規追加（既存の `.reveal-mode-note` の下、控えめな小文字リンク行）。
4. `numbers-to-memorize.html` のリード文と末尾nav を更新し、拡充した内容と導線の存在を明示。
5. `NOTES.md` に「暗記表の設計ルール」を追記——今後レッスンを拡充するたびに暗記表への数値の反映と `.see-lesson` リンクの追加を徹底するためのルール化。

## 検証

- Node スクリプトでHTMLタグの開閉バランスを確認（div/table/tr/td/span等）— 3ファイルとも OK。
- ブラウザで `numbers-to-memorize.html` を目視確認済み。

## 残課題

- `symbol-decoder.html` の技能試験セクションはレッスン16・17と候補13問マップの3方向リンクになっており、他より少し情報過多。将来的に整理してもよい。
- 暗記表はあくまで「数値」に特化しており、施工方法・器具名など**数値以外の暗記対象**（工具名、配管記号など）は `glossary.html` や各デコーダーページ側でカバーする方針を維持。
